Room Settings Details

How Settings Reach Clients

Settings are stored once under the Firebase room as settings. mpv clients receive them through the existing command poll; there is no separate settings poll.

Save writes one settings object. Reset deletes that Firebase settings object, so clients fall back to built-in defaults.

Baseline Polling

Command poll is the fixed interval, in seconds, used when adaptive polling is off. This controls how often synced mpv clients ask the helper for force sync, track sync, events, and settings.

Adaptive Polling

When adaptive polling is on, clients use Active poll briefly after sync activity, then return to Idle poll.

  • Sync turned on: active for 8 seconds.
  • Force sync received: active for 8 seconds.
  • Track sync received: active for 6 seconds.
  • Guest host-found snap or initial forced apply: active for 8 seconds.
  • Host auto force-sync after seeking: active for 8 seconds.

Normal playback does not keep clients in active polling forever. If the helper cannot be reached, clients back off up to Reconnect max.

Sync Behavior

Seek lock snaps guests back if they move farther than Seek lock gap from the projected host time.

Auto sync seek lets the host send a force sync when the host seek jump is at least Host seek gap. Seek cooldown limits how often those auto syncs can fire.
